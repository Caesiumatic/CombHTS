## What went wrong, and the fixes (cascade)
The first batch (418002) failed ~96% (558/579). Root causes, in order of discovery:
1. **Parallel ORCA crashes on Lop.** `%pal nprocs 16` → OpenMPI `opal_hwloc_topology_load`
   segfault aborts every run (the documented reason cosmors already runs serial). → set
   `nprocshared: 1` (serial ORCA). nprocs is NOT in the cache method label.
2. **config_hash mismatch.** Editing `tier2_orca.yaml` changes `tier2_config_hash` (it binds to the
   config-file bytes — intentional provenance), so the existing manifest was rejected by every task.
   → re-plan with the serial config (operational, by-design — config edits require re-planning).
3. **Login-node python is 2.7.** The `tier2-plan` step ran under system python2.7 → SyntaxError on
   the py3 CLI → no manifest → redox never submitted. → launcher/inline now `conda activate combhts`
   first (committed: `launch_tier2_fullsweep.sh`).
4. **Concurrency capped at 30.** Not slots (quota 320, smp PE 999 slots, queues had ~450 free) and
   not `max_aj_instances` — it is the scheduler's **`maxujobs 30`** (max concurrent running jobs per
   user; each array task counts as a job). **Admin-set; not user-overridable.** smp 2 / queue
   pinning / qalter across ib2+intel72smt+amd16smt all hit this wall.

## Verified working (serial ORCA)
- Monomer/solvent redox tasks succeed; e.g. BF₄⁻ anion-ox AIP = 8.352 eV (sane — BF₄⁻ is hard to
  oxidize). Small species ~10–15 min/task (2 opt+freq); 28 ok early.
- **All failures are electrolyte ions** — single-atom ions (Cl⁻/K⁺/Li⁺/Na⁺, one per solvent →
  "12 KCl") which can't be geometry-optimized, **plus** oxidized polyatomic-anion radicals
  (ClO₄•/PF₆• are unbound, ORCA opt fails). This is *expected chemistry*, not a code bug: a
  closed-shell electrolyte anion that won't oxidize is exactly what a good supporting electrolyte
  should be.

## Decision (2026-06-29): drop the electrolyte-ion redox tasks from the running array
Under the hard `maxujobs=30` concurrency cap, the 30-wide pipe is the bottleneck, and the
**121 electrolyte-ion tasks** (entity_type electrolyte_anion/cation; manifest indices 1–121, sorted
first) are its worst use — most fail and the rest are not on the scientific critical path.
`qdel 418031 -t 1-121` removed them; the **530 monomer+solvent tasks** (the f1 window + f2-vs-monomer
core) keep the pipe.

- **Why it's safe:** the §5 f2 (anion stability) term **falls back to the Tier-1 calibrated anion
  Eox** for the dropped ions (already in `tier1_all.csv`), and the refined screen handles a missing
  Tier-2 anion value. Electrolyte successes already computed (e.g. BF₄⁻ 8.35 eV) are **retained** in
  the harvest — only pending/failing tasks were dropped.
- **Impact:** redox finishes ~2.3 h sooner (121/30 × ~35 min); dimer/band-gap start sooner.
- **Reversible:** the electrolyte-ion DFT can be re-run post-presentation (`eps tier2-plan` already
  emits them) to upgrade f2 to full DFT; this is a timeline-driven deferral, not a permanent
  exclusion.
