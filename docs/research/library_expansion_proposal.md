# Library Expansion Proposal — Vetted Candidate Set Toward Directive Scale

_Proposal date: 2026-06-22 · Branch: `track/library-proposal` · Status: **PROPOSAL ONLY — NOT WIRED**_

> **This document changes nothing in the production library.** It is a planning-stage,
> web-vetted candidate list for the monomer × solvent × supporting-electrolyte chemical
> space. No row here has been added to `data/monomers.csv`, `data/solvents.csv`, or
> `data/electrolytes.csv`; no `src/`, config, or scoring code was touched. Every candidate
> carries a **SUPPORTED / CAUTION / UNSUPPORTED** tag and web-supported rationale.
>
> **Adoption is gated.** Nothing below should be wired into the library until (a) a human
> reviewer + a Deep Research pass sign off on the chemistry and SMILES, **and** (b) the
> science gates — ESW, solubility, and optical-gap calibration (STATUS.md debt #1–#7) —
> are demonstrably stable. Expanding the triad count before those gates are stable would
> multiply an uncalibrated score across ~50k triads. See §8.

## 1. Scope and guardrails

This proposal addresses **STATUS.md open debt #8**: *"the library is 36×13×16 versus the
requested roughly 80–150 × 25–35 × 20–30."* The directive target is ~80–150 monomers,
~25–35 solvents, and ~20–30 electrolytes (≈50k triads).

Guardrails honored in producing this file:

- **Docs-only.** The only file written is this one. No library CSV, config, scoring axis,
  or `src/` module was modified.
- **Dedicated branch** `track/library-proposal`.
- **Web-supported rationale** for every candidate (sources cited per row).
- **Honesty over coverage.** Uncertain SMILES are flagged at lower confidence rather than
  fabricated; non-electropolymerizable or engine-incompatible items are listed as
  `UNSUPPORTED` (documented rejects), not quietly dropped.

## 2. Methodology

Candidates were generated with a **deep-research harness** (5 parallel web-search agents,
one per chemical-space angle: thiophene/EDOT donors; furan/selenophene analogues folded in;
pyrrole/amine/carbazole; fluorene/fused/donor–acceptor; solvents; salts), followed by an
**adversarial verification pass** — one skeptical critic per category that deduplicated
against the current library, checked SMILES and coupling regiochemistry, re-tested
electropolymerizability and engine support, and finalized each tag.

**Independent post-hoc integrity check** (run locally with the project RDKit, `.venv`):

- **76/76 monomer SMILES**, **27/27 solvent SMILES**, and
  **25/25 salt cation+anion SMILES** parse and sanitize cleanly in RDKit.
- **Zero duplicates** against the current library; **zero internal duplicate structures**.
- This only certifies the SMILES are *parseable and novel to the library* — it does **not**
  certify electropolymerizability or that geometry generation + xTB will converge for every
  structure. Those remain part of the human/Deep-Research sign-off.

### Engine-support facts that drove the tags

- **xTB ALPB** is CSV-driven (`xtb_gbsa_name`) with no hardcoded allowlist, but xTB only
  accepts its **real native ALPB keywords**: `acetone, acetonitrile, aniline, benzaldehyde,
  benzene, ch2cl2, chcl3, cs2, dioxane, dmf, dmso, ether, ethanol, ethylacetate, furane,
  hexadecane, hexane, methanol, nitromethane, octanol, woctanol, phenol, thf, toluene,
  water`. A solvent with no native parameter must use an **ALPB proxy** (nearest-dielectric
  mapping) — treated here as a `CAUTION`, not clean support.
- **ORCA/Gaussian SMD** has ~180 built-in solvent names; **openCOSMO-RS pilot is currently
  acetonitrile-only**. Solvents needing a custom 7-descriptor SMD definition are flagged.
- **Continuum solvation fails** for ionic liquids, deep eutectic solvents, and BF₃·OEt₂
  (BFEE) Lewis-acid media → those are `UNSUPPORTED-for-now` regardless of chemical merit.
- **Geometry** (RDKit ETKDGv3 + xTB GFN2): Se handled (force-field typing skipped), fused
  rings and large oligomers embed; radicals/charged species use explicit charge/multiplicity
  flags (not parsed from SMILES) — so permanently charged/self-doped monomers are flagged.

## 3. Tag definitions

| Tag | Meaning |
|---|---|
| `SUPPORTED` | Clean add: chemically sound, electropolymerizable (monomers) or natively engine-supported with obtainable data (solvents/salts), low risk. |
| `CAUTION` | Add but flagged: solubility limits, side reactions, charged/self-doped species, ALPB-proxy-only solvent, ESW needs measuring, approximate coupling regiochemistry, geometry risk, or ion-pairing concerns. |
| `UNSUPPORTED` | Do **not** add now: engine cannot handle it (continuum-invalid IL/DES/BFEE, no polymerizable handle), not electropolymerizable (blocked coupling sites), or reference-only salt. Listed as a documented reject. |

Notes on the framing:

- **Monomers**: the SUPPORTED+CAUTION pool (74) lands the library at ~110, comfortably
  inside 80–150 even after sign-off attrition. SUPPORTED-only would be 61 (below range), so
  some CAUTION items are expected to be promoted after review.
- **Solvents**: the candidate pool intentionally **over-provisions**. Only `acetone` is
  `SUPPORTED` because the critic (correctly) demoted every proxy-mapped or ESW-unmeasured
  solvent to `CAUTION`. Several CAUTION solvents are nonetheless **engine-native** (chloroform,
  1,4-dioxane, ethyl acetate, methanol, ethanol have true xTB ALPB params + built-in SMD
  names) and were flagged for *chemistry/data* reasons (narrow/protic window, ESW to be
  pinned), not engine reasons. The wired set should be a **curated subset of ~12–22 additions**
  to land at 25–35, prioritizing engine-native + measured-ESW solvents.
- **Salts**: SUPPORTED-only already lands the library at 21 (in range). The CAUTION pool
  is a reserve; wiring all of it would overshoot 30, so curate.

In short: this is a **vetted candidate reservoir sized above the target on purpose**, so the
final wired library still lands in range after human/Deep-Research attrition. It is not a
"wire all of it" list.

## 9. Caveats & honesty notes

- **Tags are screening-grade, not adjudicated.** Each tag reflects one adversarial critic
  pass over web evidence, not the 3-vote-per-claim depth a final adjudication would use. Treat
  every `SUPPORTED` as "clean candidate," not "approved."

  coupling is not a simple α–α link); these mirror the existing `o-toluidine`/`diphenylamine`
  "APPROXIMATE" caveats already in the library.

  bare acceptors with no handle are tagged `UNSUPPORTED` (e.g. the PDI core).

  SaltRemover/sanitization behavior on counterions must be checked before wiring.

- **Solvent ESW values cited "vs DMFc" or in battery contexts** are not yet pinned to the
