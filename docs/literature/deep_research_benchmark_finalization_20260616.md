# Monomer Oxidation Benchmark Final Scientific Curation

## Final verdict

**Section 1 — Final verdict.** I audited **11 current repo rows** from the attached benchmark nucleus, reviewed **6 additional literature/thesis source families** beyond the current repo sources, and assembled a **strict benchmark-ready set of 14 raw rows**, all **14 calibration-eligible**, giving **14 collapsed groups** under the required key `monomer_smiles + solvent_name + label_type`. The requested target of **at least 30** strict collapsed groups was **not achieved**. Under the attached curation protocol, the limiting factors were not search effort alone, but source auditability, label hygiene, solvent-schema compatibility, and non-speculative reference handling.

The current attached benchmark nucleus is already described in the methods memo as only **11 rows from four source families**, and the attached validator logic expects exact ontology metadata and clean grouping behavior before calibration. In this final pass, **no current repo row remained benchmark-ready as-is**. The main reasons were: threshold electropolymerization labels being mixed with monomer oxidation labels, nonaqueous `SCE -> Ag/AgCl` bookkeeping that the strict policy does not accept, unresolved figure/table/page locators in the current nucleus, and one clearly inconsistent 3-hexylthiophene source. The 3-hexylthiophene paper’s webpage still shows an `Ag/Ag` reference statement tied to `TBAPF6/CH3CN`, while the monomer experiment itself uses `LiClO4` in acetonitrile and states a scan rate of `0.1 mV/s`; that combination is not clean enough for calibration. ([doi.org](https://doi.org/10.32508/stdj.v26i2.4063))

The strict final set is therefore the **14 direct-Ag/AgCl rows** already isolated in the second finalization report from the Cihaner/Önal and Camarada source families. Those rows avoid speculative nonaqueous reference conversion and preserve the crucial separation between **thiophene/acetonitrile peak-like behavior at 2.05 V** and **thiophene/acetonitrile onset-style behavior at 1.50 V**. Averaging those would contaminate the calibration target.

In the conversion expression

\[
E_{\mathrm{vs\ Ag/AgCl}} = E_{\mathrm{native}} + \Delta E_{\mathrm{ref}},
\]

the quantity \(\Delta E_{\mathrm{ref}}\) is the signed offset used to move a source-reported potential onto the benchmark Ag/AgCl scale. In the final strict benchmark below, every retained row uses **\(\Delta E_{\mathrm{ref}} = 0.00\ \mathrm{V}\)** because the source already reported on an Ag/AgCl-type scale. This is the main reason those rows survived while the current SCE, Ag/Ag⁺, and pseudo-reference rows did not.
